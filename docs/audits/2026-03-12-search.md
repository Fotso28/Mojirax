## Resume fonctionnel

### Architecture de la recherche

1. **Recherche universelle** (`GET /search/universal?q=...`) — Composant header `UniversalSearch`
   - Recherche texte immediate (<20ms) + semantique en parallele (timeout 2s)
   - Fusion : semantique prioritaire, texte en complement, deduplication par ID
   - Champs : nom, pitch, description, secteur (synonymes FR), role (synonymes FR), ville, pays, skills
   - Retourne : projets (5), personnes (5), skills (5)

2. **Recherche semantique** (`GET /search?q=...&sector=...&city=...`) — Page `/feed/search`
   - Embedding Jina AI + cosine similarity pgvector
   - Filtres : secteur, ville
   - Timeout 3s sur requetes vectorielles
   - Retourne : projets (10), candidats (10)

3. **Historique** (`GET/DELETE /search/history`) — Protege par `FirebaseAuthGuard`

### Endpoints

| Methode | Path | Guard | Rate limit | DTO | Description |
|---------|------|-------|------------|-----|-------------|
| GET | `/search/universal` | FirebaseAuthOptionalGuard | 15/60s | SearchQueryDto | Recherche universelle |
| GET | `/search` | FirebaseAuthOptionalGuard | 15/60s | SemanticSearchQueryDto | Recherche semantique avec filtres |
| GET | `/search/history` | FirebaseAuthGuard | 10/60s | - | Historique des recherches |
| DELETE | `/search/history` | FirebaseAuthGuard | 5/60s | - | Supprimer l'historique |

### Modeles Prisma
- `SearchLog` — Logs avec searchType: `UNIVERSAL`, `TEXT_FALLBACK`, `SEMANTIC_HYBRID`
- `Project` — `description_embedding` (pgvector 1024-dim)
- `CandidateProfile` — `bio_embedding` (pgvector 1024-dim)
- `FilterEmbedding` — Cache embeddings skills
