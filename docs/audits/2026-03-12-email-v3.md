## Resume fonctionnel

### Architecture

```
notify(userId, type, title, message, data)
  |-- Create Notification in DB
  |-- sendPush() [PushService] (fire & forget)
  |-- sendEmail() [EmailService] (fire & forget)
       |-- Check EmailConfig (enabled + type)
       |-- Fetch User (email, preferredLang)
       |-- Resolve template + actionUrl + variables
       |-- Compile MJML -> HTML (cached layout)
       |-- Send via Brevo API
       |-- Log to EmailLog
```

### Endpoints

| Methode | Path | Guard | Description |
|---------|------|-------|-------------|
| GET | /admin/email-config | FirebaseAuthGuard + AdminGuard | Config email |
| PATCH | /admin/email-config | FirebaseAuthGuard + AdminGuard | Modifier config email |

### Modeles Prisma

- `User` : +`preferredLang` (String, defaut "fr"), +`emailLogs` (relation)
- `EmailConfig` (nouveau) : Singleton admin
- `EmailLog` (nouveau) : Tracking emails (userId, type, status, brevoId)
- `NotificationType` : +WELCOME, +ONBOARDING_REMINDER, +MESSAGE_RECEIVED
