## Resume fonctionnel

### Ajouts

- **EmailService** : Envoi d'emails via Brevo API, fire & forget, avec config admin
- **EmailCompilerService** : Compilation MJML en HTML responsive, i18n FR/EN, echappement HTML
- **email.constants.ts** : Mappings type → template, type → actionUrl, resolution sous-templates MODERATION_ALERT
- **28 templates MJML** : 14 FR + 14 EN (welcome, application-received/accepted/rejected, moderation-published/rejected/pending, profile-published/review, document-analyzed/failed, profile-unlocked, onboarding-reminder, system)
- **i18n** : Sujets et labels FR/EN
- **EmailConfig** : Config admin singleton (enabled, enabledTypes, fromName, fromEmail)
- **EmailLog** : Tracking de chaque email envoye (userId, type, status, brevoId)
- **WELCOME** : Email de bienvenue declenche dans `syncUser()` pour les nouveaux utilisateurs
- **ONBOARDING_REMINDER** : Cron quotidien 10h, batch 50, users inactifs 48h+
- **UpdatePushConfigDto** : DTO class-validator pour l'endpoint push-config (corrige le type inline)

### Endpoints touches

| Methode | Path | Description |
|---------|------|-------------|
| GET | /admin/email-config | Recuperer la config email |
| PATCH | /admin/email-config | Modifier la config email |

### Modeles Prisma impactes

- `User` : +`preferredLang` (String, defaut "fr"), +`emailLogs` (relation)
- `EmailConfig` (nouveau) : Singleton, enabled, enabledTypes, fromName, fromEmail
- `EmailLog` (nouveau) : userId, type, to, subject, brevoId, status, error
- `NotificationType` : +WELCOME, +ONBOARDING_REMINDER

### Dependencies ajoutees

- `@getbrevo/brevo` — SDK Brevo API transactionnelle
- `mjml` + `@types/mjml` — Compilation MJML vers HTML
- `@nestjs/schedule` — Cron pour ONBOARDING_REMINDER
